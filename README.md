# コーデ日和 (Weather-Linked AI Stylist)

ユーザーの「今、その場所」のリアルタイムな環境に100%パーソナライズされた対話型AIスタイリストアプリケーション。

## 📝 プロジェクト概要
従来のコーディネート提案ツールや天気アプリは固定された地域データに基づくものが多く、ユーザーが「今どこにいて、これから何をするか」というリアルタイムな文脈（コンテキスト）に寄り添えていない点に課題がありました。
本システムは、**ブラウザ標準のGPSインフラ機能**と**グローバルな気象データベースAPI**を連携させ、取得した環境生データをFastAPI経由でLLMのプロンプトへ動的に注入（RAG）するパイプラインを自作しました。これにより、お出かけ前のユーザーへ実用性とコンテキスト精度の高い着こなし提案を可能にしています。

---

## 🛠 技術スタック (Technical Stack)

### 💻 フロントエンド (Frontend)
* **Next.js (v14+ / App Router)**: `use client` によるインタラクティブな状態管理、GPS・気象データの即時同期。
* **TypeScript**: 型安全性を担保し、コンポーネント間（Props）およびAPIレスポンスのデータ受け渡しの堅牢性を確保。
* **Tailwind CSS**: ニュアンスカラーを基調としたグラスモフィズム（背景ぼかし）UI、快適なホバーエフェクトの実装。
* **React Markdown**: LLMからの応答テキストのパース・クレンジング処理。

### ⚙️ バックエンド & AI (Backend & AI)
* **FastAPI (Python)** / **Uvicorn**: 非同期処理によるフロントエンドからのリクエスト高速ハンドリング。
* **RAG (Retrieval-Augmented Generation)**: 外部気象APIから取得した「都市名・現在気温・気候ステータス」をLLMのプロンプトへ動的にインジェクションし、ハルシネーション（嘘の回答）を排除したコンテキスト制御。

### 📡 外部API & 外部インフラ
* **OpenWeatherMap API**: 緯度経度ベースによるリアルタイムな詳細気象データの取得。
* **HTML5 Geolocation API**: ブラウザ標準の端末GPSインフラの活用。
